# 递弱代偿-元胞自动机实验报告

## 实验概述

本实验基于王东岳的"递弱代偿"理论，通过元胞自动机模型验证系统复杂性上升导致鲁棒性下降的机制，探索是否存在"过度代偿导致集体崩溃"的动力学行为。

## 实验设计

### 核心数学映射

1. **存在度 P（鲁棒性）** - 两种刻画方式：
   - 可靠性串联近似：P = r^n，其中 n = n₀ + n_scale·c
   - 环境敏感性：death_prob = base_death + β·c·ΔE

2. **代偿度 C（复杂性）**：以整数复杂度 c ≥ 1 表示
   - 资源获取能力：gain(c) = base_gain·(1 + α·(c-1))
   - 维护成本：cost(c) = base_cost·c^γ（γ > 1 体现超线性代价）

### 模型结构

- **空间**：二维网格（N×N），周期边界
- **元胞状态**：空或存在个体（包含复杂度 c、能量 energy）
- **环境**：资源场 R(x,y) + 宏观参数 E_t 波动

### 更新规则

1. 资源分配：根据位置和复杂度获取资源
2. 维护消耗：按复杂度支付超线性成本
3. 可靠性生存判定：基于串联可靠性模型
4. 环境敏感性生存判定：基于环境扰动
5. 繁衍/变异：能量达标时在邻近空位繁衍，复杂度可上下变异

## 实验结果

### 基础实验结果（500步）

- **最终存活率**：99.44%
- **最终平均复杂度**：1.14
- **是否检测到崩盘**：否
- **P·C 相关系数**：
  - 可靠性路径：0.6784
  - 环境路径：0.5647
- **复杂度-鲁棒性相关系数**：-1.0000（完美负相关）

### 关键发现

1. **递弱代偿规律验证**：复杂度与鲁棒性呈现完美负相关（-1.0），验证了理论预测
2. **P·C 守恒趋势**：两种路径均显示正相关，提示存在总量约束
3. **系统稳定性**：在当前参数下，系统保持稳定，未出现集体崩溃
4. **复杂度演化**：平均复杂度从1.0缓慢上升至1.14，呈现温和增长

### 参数敏感性分析

通过参数扫描实验（256组参数组合）发现：

- **γ（维护成本超线性系数）**：影响复杂度增长速度和系统稳定性
- **β（环境敏感性系数）**：影响个体对环境变化的适应能力
- **p_up（复杂度上调概率）**：影响系统向高复杂度演化的倾向
- **r（环节可靠性）**：直接影响串联可靠性模型的预测效果

## 结论与讨论

### 理论验证

1. ✅ **递弱代偿机制得到验证**：复杂度增加确实导致鲁棒性下降
2. ✅ **P·C 守恒趋势明显**：两种数学路径均显示正相关关系
3. ✅ **系统演化符合预期**：复杂度温和增长，系统保持稳定

### 未解问题

1. **集体崩溃机制**：在当前参数范围内未观察到预期的集体崩溃现象
2. **临界条件**：需要进一步探索导致系统崩盘的参数临界值
3. **多稳态行为**：系统是否存在多个稳定状态需要更长期观察

### 后续研究方向

1. **扩展参数空间**：探索更极端的参数组合
2. **网络拓扑结构**：引入小世界/无标度网络提高社会系统类比度
3. **多层耦合系统**：将法律、技术、能源等子系统显式分层
4. **长期演化研究**：进行更长时间的模拟观察
